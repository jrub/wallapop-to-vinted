# Architecture Decision Records

Each ADR captures a single architectural decision following Michael Nygard's format: **Context** (the forces at play), **Decision** (what we chose), **Status** (Proposed / Accepted / Superseded), and **Consequences** (the trade-offs). ADRs are numbered sequentially (`NNNN-short-title.md`), never edited after acceptance — a superseding decision gets its own ADR and updates the previous one's status.
